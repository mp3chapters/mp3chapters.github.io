Edit chapters

Chapters are managed using the same syntax also used by YouTube. Each chapter is one line, starting with the timestamp when the chapter begins (for example 03:20 or using milliseconds 14:21.320), followed by a space, followed by the name of the chapter. Chapters can also have a link and an image associated with them, see below for details.

Tip: and paste it into your episode description, which some players like Spotify use.
Advanced: Chapter URLs, images, hidden chapters

Each chapter can be assigned a link that will be shown by some players. To add a link, add the URL (starting with http:// or https://) to the end of the chapter line.

Each chapter can be assigned an image. To add an image, upload an image and add its code (for example <img-6>) to the end of the chapter line. Images in the gallery that are not assigned to a chapter are ignored. The same image can be used for multiple chapters. A chapter can get both a link and an image.

A chapter can be hidden from the table of contents (the chapter list). This can be useful to show a chapter image for some time period without showing a separate chapter. To hide a chapter, begin its name with _ (an underscore). The chapter name can also consist of only _. Note that hidden chapters are not supported by all players.

Podcast Chapter JSON format and Podlove Simple Chapters XML

There are alternatives to embedding chapters directly in the audio file, including the Podcast Chapter JSON and Podlove Simple Chapters XML formats, which can be linked to (JSON, XML) or embedded in the <item> tag (XML) of a podcast RSS feed. While most podcast players use embedded chapters, some other players (notably Spotify) only support time stamps in the episode description or podlove chapters.
